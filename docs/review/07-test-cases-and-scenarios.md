# 测试用例与场景（论文第5章可直接引用）

## 1. 功能主链路

### TC-FUNC-01 用户注册登录
- 前置：无
- 步骤：注册 -> 登录 -> 调用 `/user/auth`
- 预期：返回 token 与用户信息

### TC-FUNC-02 商品发布与浏览
- 前置：已登录用户
- 步骤：发布商品 -> 首页查询商品列表
- 预期：商品可见，状态为 ON_SALE

### TC-FUNC-03 预约交易双确认
- 前置：存在 ON_SALE 商品，买卖双方账号
- 步骤：买家预约 -> 卖家确认 -> 买卖双方确认完成
- 预期：订单状态 1->2->3/4，最终商品 SOLD

### TC-FUNC-04 订单取消与回退
- 前置：订单处于 1/2/3
- 步骤：买家或卖家取消
- 预期：订单变 5，商品在无活跃预约时回 ON_SALE

## 2. 异常链路
- 重复预约同商品
- 订单已完成后重复确认
- 订单已取消后再取消

## 3. 权限场景

### TC-AUTH-01 未登录访问受限接口
- 请求：`/orders/query`, `/message/query`
- 预期：认证失败

### TC-AUTH-02 普通用户访问管理员接口
- 请求：`/dashboard/staticCount`
- 预期：权限不足

### TC-AUTH-03 横向越权更新商品
- 请求：普通用户更新他人商品
- 预期：被拒绝

## 4. 安全场景

### TC-SEC-01 路径穿越
- 请求：`/file/getFile?fileName=../application.yml`
- 预期：400

### TC-SEC-02 token 篡改
- 请求：篡改 token 后访问受限接口
- 预期：认证失败

## 5. 性能与工程场景
- 前端构建耗时、包体积统计
- 关键接口（商品查询、订单查询）响应时间采样
